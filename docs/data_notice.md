# Data Sanitization Notice

## Policy

This repository is a **public, reproducible research artifact**. All data included herein is either:

1. **Synthetic** — generated by Gazebo VRX simulation (wave models, virtual sensors)
2. **Mock** — fabricated sensor logs for unit testing (see `data/mock/`)

## What Is NOT Included

The following real-world data has been **deliberately excluded**:

| Data Type | Reason |
|-----------|--------|
| Real camera footage (`.mp4`, `.avi`, image sequences) | Contains identifiable vessels, shorelines, personnel |
| Raw GPS logs (`.nmea`, `.ubx`) | Reveals operational locations |
| IMU recordings (`.csv`, `.bag`) | Contains calibrated sensor fingerprints |
| LiDAR point clouds (`.pcd`, `.las`) | Reveals surveyed environment geometry |
| Trained model weights (`.pt`, `.pth`, `.onnx`) | Proprietary training data in weights |
| ROS bag files (`.db3`, `.mcap`) | All of the above combined |

## Reproducibility

All results claimed in this project are reproducible using **only** the included Gazebo simulation:

```bash
docker compose up
# or
ros2 launch usv_bringup multi_boat_sim.launch.py
```

The simulation includes:
- Wave dynamics (amplitude, period, direction)
- Wind forces
- Water current
- Buoyancy and drag
- Camera and IMU sensor models
- GPS with configurable noise

## Verification

To verify that the simulation produces equivalent results to the original experiments:

1. Run the test suite: `pytest tests/ -v`
2. Compare trajectory logs from `data/mock/sensor_log.csv` against simulation output
3. All control algorithm tests (LOS, PID, Boids, ORCA) use deterministic inputs and have known correct outputs
